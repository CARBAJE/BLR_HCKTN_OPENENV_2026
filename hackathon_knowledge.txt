1. PROJECT IDENTITY
-------------------
Name: ReactOS (OSaaS — OS as a Service)
Domain: Simulated operating system for AI agent training and evaluation
Real-world task: Desktop OS navigation (opening apps, installing software,
    managing files, configuring settings)
Benchmark name: ReactOS
Environment type: Interactive GUI navigation via DOM elements


2. INFERENCE SCRIPT RULES (STRICTLY ENFORCED)
----------------------------------------------
File: inference.py (MUST be in root directory of the project)
SDK: openai Python SDK ONLY (no alternative SDKs or direct HTTP calls)

Environment variables (MUST read these):
  API_BASE_URL  — LLM endpoint          (default: "https://api.openai.com/v1")
  MODEL_NAME    — model identifier       (default: "gpt-4o-mini")
  HF_TOKEN      — API key               (REQUIRED, no default, raise ValueError)

Client initialization:
  client = OpenAI(base_url=API_BASE_URL, api_key=HF_TOKEN)


3. STDOUT FORMAT (CRITICAL — ANY DEVIATION = INCORRECT SCORING)
----------------------------------------------------------------
Exactly three line types, in this order:

  [START] task=<task_name> env=<benchmark> model=<model_name>
  [STEP] step=<n> action=<action_str> reward=<0.00> done=<true|false> error=<msg|null>
  [END] success=<true|false> steps=<n> score=<score> rewards=<r1,r2,...,rn>

Formatting rules:
  - One [START] line at episode begin
  - One [STEP] line per step, immediately after env.step() returns
  - One [END] line after episode completes, ALWAYS emitted (even on exception)
  - reward, score, and all items in rewards: strictly 2-3 decimal places
  - done and success: lowercase booleans (true or false)
  - error: raw error string, or null if none
  - All fields on a single line, no newlines within a line
  - score must be in [0, 1]

Example:
  [START] task=open_terminal env=ReactOS model=gpt-4o-mini
  [STEP] step=1 action=CLICK(Start Button) reward=0.03 done=false error=null
  [STEP] step=2 action=CLICK(Terminal) reward=1.00 done=true error=null
  [END] success=true steps=2 score=1.00 rewards=0.03,1.00


4. ENVIRONMENT SPEC (OPENENV COMPLIANCE)
-----------------------------------------
Required endpoints (via FastAPI):
  POST /reset        — reset episode, return initial observation
  POST /step         — execute action, return observation + reward + done
  GET  /state        — return current state
  GET  /health       — return {"status": "healthy"}
  GET  /schema       — return JSON schemas for action/observation

openenv.yaml format:
  spec_version: 1
  name: react_os
  type: space
  runtime: fastapi
  app: server.app:app
  port: 8000

Pydantic models required:
  - Action subclass (with action type and target)
  - Observation subclass (with done, reward, DOM state, instruction)
  - State subclass (with episode_id, step_count, plus env-specific fields)


5. TASKS (MINIMUM 3, EASY -> MEDIUM -> HARD)
----------------------------------------------
Each task defines:
  - A concrete objective the agent must accomplish
  - A programmatic grader that scores performance in [0.0, 1.0]
  - Clear, deterministic success/failure criteria

Our tasks:
  EASY   — "Click the Start button"
            Target: Start Button (single CLICK)
            Score: 1.0 if clicked, 0.0 otherwise
            Expected steps: 1

  MEDIUM — "Open the Terminal"
            Target: Terminal (open via Start menu)
            Score: 1.0 if Terminal window opens, partial for start menu opened
            Expected steps: 2-3

  HARD   — "Install Python and verify in terminal"
            Multi-step: open Downloads folder, find installer, run it,
            click Install Now, open terminal, verify python command
            Score: partial credit for each milestone reached
            Expected steps: 6-15


6. REWARD FUNCTION DESIGN
---------------------------
Multi-component reward in [0, 1]:

  element_score x action_score  (0.0 or ~1.0) — correct element + action type
  visibility_bonus              (0.05)         — target became visible after action
  state_change_bonus            (0.03)         — OS state actually changed
  proximity_reward              (0-0.1)        — clicked near target but missed
  exploration_bonus             (0-0.12)       — new icons/elements appeared

Rules:
  - Desktop icons REQUIRE DOUBLE_CLICK to open
  - All other elements use CLICK
  - Final reward = min(1.0, sum of all components)
  - Done condition: element_score * action_score >= 1.0 OR step >= max_steps


7. ACTION SPACE
----------------
The agent sends:
  {
    "action": "CLICK" | "DOUBLE_CLICK",
    "node_idx": <int>  (0-based index into the DOM element list)
  }

The environment resolves coordinates from the DOM element at node_idx.


8. OBSERVATION SPACE
---------------------
The environment returns:
  {
    "dom": [
      {"text": "Start Button", "type": "button", "x": 0.034, "y": 0.972},
      {"text": "Desktop icon: This PC", "type": "icon", "x": 0.5, "y": 0.3},
      ...
    ],
    "instruction": "Open the Terminal"
  }

Element types: button, icon, menuitem, window, taskbar, element, tab, input
Coordinates: normalized to [0, 1] (x = pixel_x / 1280, y = pixel_y / 720)


9. OS STATE MACHINE
--------------------
The simulated OS supports:
  - Window management (open, close, focus, minimize, maximize)
  - Start menu (toggle, close)
  - Desktop icons (This PC, Downloads, System Settings, Recycle Bin, + random files)
  - File Explorer (browsing directories, showing files)
  - Terminal (command execution: echo, dir/ls, python, path, set, cd, cls)
  - Python installer (Install Now, Customize Installation, Add to PATH)
  - System Settings (System tab, Environment Variables tab)
  - Clipboard operations
  - Environment variables (PATH, USERPROFILE, TEMP, etc.)

Visual randomization on each reset:
  - Taskbar position: top, bottom, left, right
  - Accent color: 10+ options
  - Font family: 4 options
  - Desktop background: 8 colors + 5 wallpaper images
  - DPI scale: 0.85, 1.0, 1.15, 1.25
  - Random file names on Desktop and Downloads
